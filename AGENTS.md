# TribeLog Project Memory

Future agent sessions should read `docs/STRATEGIC_MOAT_AND_IP.md` before making product, growth, monetization, roadmap, IP, or positioning recommendations.

Persistent strategy context:

- TribeLog should become the operating system for small-group fitness accountability.
- The defensible moat is community graph, creator/admin operating system, proprietary progress data, partnerships, trust, and brand/IP protection.
- Generic features such as challenges, streaks, points, leaderboards, onboarding, Health sync, share cards, and referrals are copyable unless they feed one of the moat loops.
- Patent exploration should focus only on specific technical inventions, such as activity normalization, anti-cheat scoring, privacy-preserving group benchmarks, dynamic challenge generation, or drop-off/comeback systems.
